이번에 SQLD(SQL 개발자) 자격증을 취득하고 와서 준비 과정을 정리해 보려 합니다.  
준비 중이신 분들께 조금이나마 참고가 되었으면 합니다.

---

## 1. 시험 정보

SQLD는 **데이터 모델링의 이해**와 **SQL 기본 및 활용** 두 과목으로 구성되어 있습니다.  
총 50문항이며, 각 과목이 **40점 이상**이고 전체 **평균 60점 이상**이면 합격합니다.

| 과목 | 구성 |
|------|------|
| 제1과목: 데이터 모델링의 이해 | 제1장 데이터 모델링의 이해 · 제2장 데이터 모델과 SQL |
| 제2과목: SQL 기본 및 활용 | 제1장 SQL 기본 · 제2장 SQL 활용 · 제3장 관리 구문 |

<div style="padding: 16px 20px; border-left: 4px solid #8B6F47; background: #FBF5EC; border-radius: 0 8px 8px 0; margin: 24px 0;">
  <p style="margin: 0 0 5px; font-weight: 700; color: #5A3E20; font-size: 0.93rem;">참고 — 시험 범위</p>
  <p style="margin: 0; font-size: 0.88rem; color: #4A3828; line-height: 1.75;">제3과목(SQL 고급활용 및 튜닝)은 SQLP 전용 범위입니다. SQLD만 준비하신다면 해당 범위는 보지 않아도 됩니다. 제2과목 제3장(관리 구문) 역시 최근 개정 이후 출제 비중이 낮아지는 추세이므로, 시간이 부족하다면 1, 2장을 우선 학습하는 것을 권장합니다.</p>
</div>

## 2. 시험 준비

| 항목 | 내용 |
|------|------|
| 준비 기간 | 약 1~2주 |
| 사용 교재 | SQL 자격검정 실전문제 (한국데이터산업진흥원) |
| 학습 방법 | 교재 문제 풀이 + 모르는 개념은 AI 검색 보충 |

컴퓨터공학과 재학 중이고 DB 수업을 통해 SQL을 한 번 접한 상태였기 때문에 짧은 기간으로 준비가 가능했습니다.  
SQL이 전혀 처음이신 분들은 기본 문법부터 먼저 익히고 교재를 보시는 것이 훨씬 수월할 것 같습니다.

모르는 개념이 나올 때마다 ChatGPT, Gemini를 활용해서 빠르게 보충하는 방식으로 공부했습니다.  
시간 관계상 교재 문제를 전부 풀지는 못했고, 틀린 문제 위주로 반복하는 전략을 택했습니다.

**교재 구성 (SQL 자격검정 실전문제)**

- 제1과목 데이터 모델링의 이해 — 총 52문제
- 제2과목 SQL 기본 및 활용 — 총 124문제
- 전체 합계 — 176문제 (제3장 제외 시 170문제)

이 교재는 실제 시험 문제와 유사도가 높기로 유명합니다.  
한 번 풀고 끝내기보다는 오답 위주로 **2~3회독**하는 것이 합격의 핵심입니다.

## 5. 합격 팁

**교재 회독이 핵심입니다.**  
문제를 처음부터 전부 푸는 것보다, 틀린 문제만 골라서 2~3회 반복하는 것이 훨씬 효율적입니다.  
이 교재는 실제 시험과 유사한 문제가 많아서, 완전히 자기 것으로 만들면 합격 가능성이 크게 높아집니다.

모르는 개념은 AI를 적극 활용하세요.  
ChatGPT나 Gemini에 개념을 질문하면 빠르고 정확하게 이해할 수 있어서, 두꺼운 이론서 없이도 충분히 보완이 됩니다.

SQL을 처음 접하는 분이라면 기본 SELECT 문법부터 먼저 손에 익힌 뒤 교재를 보시는 것을 권장합니다.  
기초 없이 바로 문제 풀이에 들어가면 막히는 부분이 많아 오히려 시간이 더 걸릴 수 있습니다.
